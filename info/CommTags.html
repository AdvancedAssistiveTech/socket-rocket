Reference Sheet: Socket Message Backslash Tags

Brief

Rocket makes use of socket connections to send communications encoded in the UTF-8 standard between connected machines. These connections can be divided into two categories: user-sent and program-defined. user-sent messages are displayed on the server dashboard as sent, with no other handling undertaken by the program other than the identifying of these strings as plain text. program-defined strings are issued via the socket to indicate that special data is being carried with regard to operations the program is carrying out, such as the upload or download of files, and are prefaced with backslashes to indicate the nature of the data being transferred.
Note that user-sent messages are still prefixed, but the rest of their content is unaffected

Taglist and functions

Tag Code Function
\a ACKNOWLEDGEMENT The ACKNOWLEDGEMENT tag is used by Rocket to confirm that requests have arrived at the endpoint
\t TEXT The TEXT tag is used by Rocket to preface user-sent strings, which are then not processed any further by the program before being displayed in the message area of the dashboard
\r DOWNLOAD_REQUEST The DOWNLOAD_REQUEST tag is used by Rocket to indicate a message that contains a file location. The application will open a dedicated temporary socket using the address of the remote machine and the dedicated port 2001 and then transfer the file over the socket
\d DOWNLOADABLE_FILE The DOWNLOADABLE_FILE tag is used by Rocket to preface strings containing the particulars of a DownloadableFile object: the name, location, and size in bytes of the file in question
